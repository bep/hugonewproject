# Design Brief: Hugo Template (whole-system)

> Status: locked 2026-04-26. Feeds `/impeccable:impeccable` and `/impeccable:craft`. Pair with `.impeccable.md` for project-level design context.

## 1. Feature Summary

The default Hugo template shipped by `hugonewproject`. A complete site system — typography, tokens, palette, layouts for home/section/single/about, header + footer — engineered to look intentional for a small site (5–30 posts, 1–3 sections) with no further design work, and to remain coherent under user customization through palette + spacing tokens.

## 2. Primary User Action

A *visitor* lands on the home page and within one second feels: **"this person has taste; I trust this."** Typography and spacing carry that feeling. The author should not have had to design anything to earn it.

## 3. Design Direction

**Modern technical**, executed as `.impeccable.md` describes: confident sans body, controlled hierarchy, slight density, no widgets or chrome. Personality from precision.

**System fonts as a real palette:**

- Body: `system-ui` (San Francisco / Segoe / Roboto). The workhorse.
- Eyebrows, metadata, dates, captions, code: `ui-monospace`. This is where the "technical" voice lives — small, lowercased, slightly tracked monospaced text doing structural work that a serif might do in editorial.
- `ui-serif`: held in reserve. Not used unless a reason emerges.
- Weight contrast does the heavy lifting (e.g., 400 body / 600 headings / 500 small-caps eyebrows).

**Light is the default mood.** Warm-tinted near-white background (~oklch 99% with a hint of brand hue), tinted near-black text (~oklch 18% same hue family). Dark variant must read as the same design inverted, not a different design.

**Density:** lists and metadata are tight; reading prose is not. Earn density where the content rewards it; give prose room to breathe (max ~68ch line length, line-height ~1.6).

## 4. Layout Strategy

A single, full-width column on the page — no sidebar, no asymmetric grid, no card grid by reflex. Width is controlled by a `--measure` token (~`min(72ch, 100% - 2*gutter)`). The page body sits left-aligned-but-centered (the column is centered; content within it is left-aligned). Vertical rhythm carries hierarchy.

**Header (every page):**
- Site title left (set in body weight, slightly larger, treated as a wordmark — no logo, no tagline here)
- Inline section links right (`site.Home.Sections`)
- Sits in the same width as the content column. No bottom border by default — separation comes from spacing, not lines.

**Home (`home.html`):**
- Site title + one-line description from `site.Params.description` set as a small intro block. Description in body weight, monospace metadata showing post count or current year above as an eyebrow.
- Then a single posts list (recent posts from the first/main section), no thumbnails by default.
- Each list row: date in monospace eyebrow style above (or inline depending on width), post title, optional one-line summary.
- Asymmetric rhythm: more space above the title block, tighter rhythm within the list, generous space between list items at first scroll, can compress further down.

**Section list (`/blog/` etc.):**
- Page title (the section's title), short description if present, full posts list.
- Same row treatment as home.
- Reuse the home list as a partial.

**Single page (`page.html`):**
- Date eyebrow (monospace), then post title (large, tight tracking, weight 600).
- Optional featured image immediately under the title, full column width, with a monospace caption beneath if present.
- Body prose at the measure (~68ch).
- In-post headings (h2/h3) reuse the type scale; h2 gets generous space above, h3 tighter.
- Inline images: full column width by default, with caption support via `figure` / `figcaption`.

**About / static pages:** identical to single, minus the date and featured image affordances.

**Footer:**
- Single muted line, body weight, no border. Year + simple statement. (Already in `baseof.html` — refine, do not redesign.)

## 5. Key States

- **Default home** (5+ posts in main section): everything as designed.
- **Empty home** (0 posts in main section): the intro block remains; in place of the list, a single muted line — *"No posts yet."* — in monospace eyebrow style. No empty card. No CTA.
- **Single section** (no other sections): header has only the site title; no nav whitespace held open.
- **Multi-section** (2–3 sections): inline section links in header. If they wrap on small viewports, drop to their own row below the title.
- **Long titles**: titles wrap to up to 3 lines without breaking layout. No truncation by default.
- **Dark mode**: triggered by `prefers-color-scheme: dark`. Background ~oklch 14% with hint of brand hue, text ~oklch 92%. Line-height +0.05 on body to compensate for light-on-dark perceptual weight.
- **No featured image**: post layout still works — title sits where the image would have been. No empty image placeholder.

## 6. Interaction Model

A reading-first site has very little interaction surface. Resist adding any.

- **Links:** body links underlined with subtle offset (`text-underline-offset: 0.2em`, `text-decoration-thickness: 1px`). Hover swaps to a stronger underline. Color stays text color by default; primary accent only on hovers and on the section nav for the current section.
- **Section nav:** current section gets the accent color (or a small mark) — visible but unfussy.
- **Header nav:** no dropdown, no mobile burger. Inline nav remains inline; if it wraps, it wraps. With 1–3 sections this never overflows.
- **Focus:** real focus rings (`outline: 2px solid ...; outline-offset: 2px`). Not removed. Built from the accent color.
- **Motion:** none by default. No fade-ins, no scroll reveals. The design earns "feels good" through static composition, not motion. (`prefers-reduced-motion` is therefore trivially satisfied.)

## 7. Content Requirements

**Tokens (the public customization API):**

```
Palette (light) — set via site.Params.style.palette → CSS vars via hugo:vars
  --bg            warm-tinted near-white
  --fg            tinted near-black (text)
  --muted         tinted mid-gray (metadata, captions)
  --rule          1px hairline color
  --accent        the only color the user typically tunes per palette preset
  --accent-fg     text color for use on accent backgrounds (rare, derived)

Palette (dark) — same tokens, inverted lightness, same hue family
  Lives under [params.style.palette.dark]

Spacing (4pt scale)
  --space-2xs (4px), --space-xs (8px), --space-sm (12px), --space-md (16px),
  --space-lg (24px), --space-xl (32px), --space-2xl (48px), --space-3xl (64px),
  --space-4xl (96px)

Typography
  --font-sans (system-ui stack)
  --font-mono (ui-monospace stack)
  --font-serif (ui-serif stack — defined but unused by default)
  Type scale: 0.78rem, 0.875rem, 1rem, 1.125rem, 1.375rem, 1.75rem, 2.25rem, 3rem
  Body line-height 1.6, heading line-height 1.15

Layout
  --measure       min(68ch, 100% - 2*var(--gutter))
  --gutter        clamp(1rem, 4vw, 2rem)
```

**Palette presets (5 mood-based names — recommendation):**

1. **calm** — warm off-white background, muted slate-blue accent. Default.
2. **warm** — cream / paper background, terracotta accent.
3. **cool** — pale blue-gray background, deep teal accent.
4. **bold** — true near-white background, strong red-orange accent. Higher contrast.
5. **soft** — pale rose-gray background, dusty plum accent.

Each ships with a dark variant in the same hue family — same mood, inverted lightness, slightly reduced chroma at the dark end per OKLCH guidance.

**Microcopy:**
- Empty posts list: *"No posts yet."*
- Footer: existing line — keep `All rights reserved` or replace with neutral phrasing; see Open Questions.
- Image captions: optional; rendered only if present in shortcode/figure markup.

**Dynamic content ranges to design for:**
- Site title: 1–4 words, up to ~30 characters.
- Site description: one line, up to ~80 characters.
- Section names: 1–2 words.
- Post titles: 3–12 words; design must accommodate a 3-line wrap without breaking.
- Posts in main section: 0–30 typical, up to ~200 must remain readable (no pagination required for prototype).

## 8. Recommended References

For implementation, consult (in order of relevance):

- `reference/color-and-contrast.md` — OKLCH palette construction, neutral-tinting against accent hue, light/dark companion derivation.
- `reference/spatial-design.md` — single-column rhythm, fluid gutter, asymmetric vertical rhythm in lists.

Skip motion and interaction-design references — neither is load-bearing here.

## 9. Open Questions

To resolve during implementation:

1. **Code blocks, blockquotes, tables, horizontal rules** — not first-class per discovery. Recommendation: ship sensible browser-aligned defaults (Chroma "monokailight"/"monokai" for code, using monospace + the muted token; quotes as left-margin-shifted italic body; tables as plain bordered with hairline rules; hr as a centered short rule). Document them as "supported, not designed" so a downstream user knows where to extend.
2. **Palette preset count** — five recommended; could be 4 (drop "soft") or 6 (add another).
3. **Footer copy** — keep `© year. All rights reserved.` or soften. Tone call.
4. **Site title typography** — recommend body weight at slightly larger size (no all-caps, no logo); revisit during implementation if a more wordmark-like treatment (small caps, monospace) reads better at the actual size.
5. **First section as "the posts section"** — home shows recent posts from `(index .Sections 0)` per existing template. Stays. Multi-section sites: the first section by weight becomes the home feed; other sections still link in nav.
