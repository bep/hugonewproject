## Design Context

### Project

`hugonewproject` is a Hugo site scaffolder CLI. It generates a new Hugo project from a template. The output of *this* repo is the template itself — what gets rendered when a user runs `hugonewproject --section blog --page about --palette pastel`.

The template is the deliverable. Every design decision must hold up across a wide range of downstream sites built on top of it.

### Users

Two distinct audiences:

1. **The site author** running the CLI: developer or technically-comfortable creator who wants a Hugo project that looks good *immediately* and can be customized with a few CSS tokens. Likely an indie hacker, technical writer, small project owner, or solo creator.
2. **The site visitor** reading the generated site: anyone — readers of a blog, visitors to a small project page, etc. The default template must read clearly to a general audience.

The job for both: get from `hugonewproject ...` to a site that "looks like the author has taste" without needing further design work.

### Brand Personality

Three words: **confident, considered, undecorated.**

Modern technical sensibility — the same instinct that makes good API documentation or a well-laid-out indie hacker landing page feel pleasant. Tight rhythm, clear hierarchy, slight density. No decoration for decoration's sake. The personality comes from precision, not flourish.

### Aesthetic Direction

**Modern technical.** Confident sans-serif body, controlled hierarchy, slightly denser spacing than a typical "magazine" template. Closer to a well-made docs site or a small studio's project page than to Medium or Substack.

**Primary site type:** general-purpose — must work for personal blog, small project landing, portfolio, or notes site without optimizing for any one of them.

**Default theme:** light, with a dark variant honored via `prefers-color-scheme: dark`.

### Anti-references

The template must explicitly NOT look like the **generic Hugo starter theme**:

- No left sidebar with site title + nav
- No dated centered-header-with-tagline
- No blue underlined links on white
- No gray rounded cards in a uniform grid
- No "Recent Posts" / "Tags" / "Categories" widget pattern
- No "powered by Hugo" footer styling

Per impeccable absolute bans (already enforced):

- No `border-left` colored stripes on cards/callouts
- No gradient text
- No glassmorphism or purple-blue gradient hero sections

### Hard Constraints

- **No external fonts.** System font stacks only (`system-ui`, `ui-serif`, `ui-monospace`).
- **No external CSS/JS frameworks.** Plain CSS, plain JS.
- **No external assets.** No remote images, icons, or fonts loaded over the network.
- **CSS-template binding must stay legible.** A user opening the template should immediately see how a layout connects to its CSS.
- **Hugo `css.Build` palette injection.** Palette tokens come in via `site.Params.style.palette` → CSS vars (`hugo:vars` import). This is the customization seam — design around it.
- **Customizable via tokens.** A user changing `--space-md` or a palette value should get a coherent result, not a broken layout.

### Design Principles

1. **Earn density.** Modern technical means a *little* tighter than editorial — but only where the content rewards it. Lists are dense; reading prose is not.
2. **System fonts, intentionally.** Treat `system-ui` / `ui-serif` / `ui-monospace` as a real palette, not a fallback. Pair them deliberately (e.g., monospace for eyebrows and metadata, sans for body, serif reserved or unused).
3. **Tokens are the API.** The palette and spacing scale are the public surface for downstream users. Keep them small, semantic, and obvious.
4. **No widgets, no chrome.** No sidebars, no card grids by reflex, no decorative dividers. Structure comes from typography and spacing.
5. **Light is the default mood.** Warm-tinted near-whites, true-but-tinted near-blacks for text. Dark variant must feel like the same design, not a different one.
6. **Customization should reward small moves.** Changing one palette value should noticeably shift the feel of the site without breaking it.
